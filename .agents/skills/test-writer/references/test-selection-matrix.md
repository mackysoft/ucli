# Test Selection Matrix

## 基本優先順位
1. Small を優先する。
2. I/O境界やプロセス境界がある場合のみ Medium を追加する。
3. Large は、他サイズで契約を担保できない場合のみ追加する。

## 変更タイプ別の必須観点
| 変更タイプ | 必須観点 | 推奨テスト種別 |
| --- | --- | --- |
| バグ修正 | 失敗再現、回帰固定、境界値 | 回帰テスト（先にRed） |
| 公開API/CLI変更 | 正常系、異常系、互換性 | 契約テスト |
| データ契約変更 | スキーマ整合、順序性、互換性 | 契約テスト、ゴールデン（必要時） |
| パーサ/正規化/シリアライズ | 不変条件、壊れた入力、巨大入力 | プロパティテスト、軽量ファズ |
| リファクタ | 挙動不変、既存契約維持 | characterization + 既存テスト強化 |
| 並行性/非決定性 | 決定性、タイムアウト、順序 | 決定化後の再現テスト |

## 補足
- 1ケース1主張を守る。
- 同型ケースはパラメータ化して統合する。
