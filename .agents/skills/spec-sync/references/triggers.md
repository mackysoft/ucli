# Spec Sync Triggers（同期が必須になる条件）

次のいずれかが起きたら、実装を続ける前に spec-sync を実行する。

## What（変更内容）が変わった
- 追加機能が増えた・減った
- 既存挙動の変更点が増えた・減った

## How（技術方針）が変わった
- 設計判断（責務分割、依存方向、データ構造、アルゴリズム）が変わった

## Acceptance Criteria が変わった
- 受け入れ条件を追加・削除・修正した
- テスト観点が変わった

## スコープが変わった
- 元々スコープ外だったものがスコープ内になった（または逆）

## リスクや未確定事項が顕在化した
- “決めきれていない前提” が増えた
- 実装を始める前提として確認事項が発生した
