# 翻訳とローカライズ

## 意味保持
- 事実、主張、条件、責任主体、時制、不確実性を保つ。
- 原文の断定の強さを、訳文で強めたり弱めたりしない。
- 原文にない背景、補足、評価を足さない。
- 原文の曖昧さが判断に必要な場合は、曖昧さを消さない。

## 語彙
- コード、コマンド、ファイルパス、識別子、設定キー、製品名、固有名詞は原文の表記を保つ。
- 定訳、既存文書、対象分野の慣用語がある場合は、それに合わせる。
- 未確定の訳語を、術語として定着しているように扱わない。
- 同じ概念には同じ訳語を使う。

## 媒体適合
- 見出し、リスト、表、コードブロック、リンク、プレースホルダーを保つ。
- 翻訳先の言語で自然な語順にする。
- 読者や公開範囲が変わる場合だけ、説明の粒度と敬体、常体、敬称を調整する。
- ローカライズでは、意味が変わらない範囲で日付、単位、表記、呼称を読者に合わせる。

## 確認事項
- 訳語の選択で設計判断やブランド判断が必要な場合は、確認が必要なこととして残す。
- 原文の意味を確定できない場合は、訳文で断定せず、解釈候補を分ける。
