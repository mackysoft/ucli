# 日本語専門原稿

## 適用範囲
日本語の専門記事、長文解説、書籍調の原稿を整えるときに使う。
この参照は、事実性、文章構造、簡潔さ、専門文書、日本語の各規則を組み合わせるためのプロファイルである。

## 整形
- 一文ごとに改行する。
- 段落の区切りは空行で示す。
- コード、数式、表、ログ、設定ファイルの断片は、媒体の規則に従って示す。
- 用語の由来や定式化の名称など、本筋から一段外れる補足は脚注へ降ろす。
- 定義や分類の列挙は箇条書きで示してよい。
- 定義される用語は太字にする。
- 用語を初めて定義または導入するときも、その語は太字にする。
- 導入済みの語を話題として指すとき、引用、通称には「」を使う。
- 用語とその定義を並べる箇条書きは、全角コロンで書く。

## 原稿の流れ
- 読者が得る結論または理解を先に置く。
- 背景、問題、判断、方法、確認、限界を順に追える形にする。
- 節ごとに主題を一つに絞る。
- 章や節をまたいで同じ説明を繰り返さない。
- 反論や疑念を処理してから結論を置く。
- 前方参照は、論証が一段落した位置に置く。

## 粒度
- 原稿として必要な説明は削らない。
- ただし、仕様書、用語集、リファレンス、作業ログに任せるべき詳細を本文へ持ち込まない。
- 定義、制約、判断理由は本文に残し、単なる操作手順は必要な範囲に絞る。
- 後で参照しない固有名、時刻、細かい数値、識別子は出さない。
- 議論に必要な具体は残す。

## 表記
- 一文が複数の判断を含む場合は分ける。
- 重要語の強調は最小限にする。
- 比喩や強い言い切りで、専門的な不確実性を隠さない。
- 日本語の地の文と見出しでダッシュ類を使わない。
- 日本語の並列で中黒を使わない。
- 見出しやコラム見出しに、区切り線で二要素を詰め込まない。
- 範囲記号、英語の複合語、コードブロック、書誌情報は、媒体の表記規則に従う。

## 視点
- 結果の羅列や受動態だけで流さず、必要な場面では行為者を主語にした動作の連なりで書く。
- 架空の人物設定を、意味なく冠しない。
- 論証の中で読者を「あなた」と呼ばず、役割名で書く。
- 二人称の呼びかけは、場面への導入や章、文書の結びなど、限られた要所にとどめる。
- 対象を指す語は具体的に選ぶ。
- 定式化した術語は、以後もその語で通す。
- 人物そのものに言及するときは原綴りを優先し、定着した概念名では通称を使ってよい。

## 仕上げ
- 導入、本文、まとめで同じ結論を同じ言葉で繰り返さない。
- 読者に残す判断、制約、次の行動を最後に整理する。
- 原稿の完成度を上げるために、確認していない専門的事実を補わない。
- 論点を増やさない空句、作業姿勢だけの宣言、空虚な強調を削る。
- 例が作為的に見えうる場合は、読者の疑念を先回りして認め、根拠を短く添える。
- その根拠は、著者の断定ではなく、読者が照合できる一般的事実や通説に置く。
