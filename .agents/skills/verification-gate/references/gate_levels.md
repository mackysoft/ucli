# Gate Levels（検証の強さ）

## Smoke（挙動確認・短時間）
- `Run` 判定になった format を最小範囲で実行
- `Run` 判定になったテストを最小範囲で実行

用途:
- 変更直後の挙動確認
- 小さな変更の短時間確認

## Standard（PR前の基本）
- `Run` 判定になった format（apply + verify）
- `Run` 判定になったテスト（影響範囲）＋必要なら追加テスト

用途:
- 通常のPR前

## Full（最終/高リスク）
- Standard の全て
- `Run` 判定になったテストを広く実行（原則フル）

用途:
- 高リスク変更
- 重要フロー変更
- 最終確認
