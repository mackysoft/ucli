# issue-writer

## 目的
与えられたIssue構造、本文入力、文書前提に従って、実装者が作業範囲を判断でき、レビュー者が完了状態を判定できるGitHub Issue本文を作る。

## 使うタイミング
- GitHub Issue本文を作成、更新、点検する。
- 用意されたIssue構造と本文入力から、起票または本文更新まで進める。
- 既存Issueの本文を、目的、作業範囲、完了条件、依存関係が分かる形に整える。

## 入力確認
本文を書く前に、入力元にかかわらず次を確認する。

- 文書前提（対象、読者、媒体、言語、出力形式）
- 本文形式
- Issue種別（`single`、`parent`、`child`）
- タイトル案
- 目的
- 作業範囲または全体の完了条件
- 依存関係
- 参照元
- 親子関係

Issueを作成または更新する場合は、対象リポジトリと作成または更新の指定も確認する。

本文形式の指定がない場合は、このスキルの本文構成を使う。
本文の作成に必要な項目、または作成と更新に必要な指定が不足している場合は、不足項目を報告に含める。
本文項目が互いに矛盾する場合は、矛盾箇所を報告に含める。
Issue構造、親子関係、Pull Requestで完了できる粒度は本文作成の前提として扱う。

## 文書前提
Issue本文の文章前提は `$writing` に従う。
媒体は GitHub Issue として扱う。
Issue本文の読者は、Issueを読んで実装、レビュー、議論、完了判定を行う人を中心に置く。
タイトル、見出し、本文、箇条書きは同じ文章前提に揃える。

## 本文構成
本文には、目的、作業範囲または全体の完了条件、依存関係を入れる。
検証と参照は、そのIssue固有の情報がある場合に入れる。

### `single`

```md
## 目的
<このIssueで到達する状態>

## 作業範囲
- <変更対象>
- <変更対象>

## 完了条件
- <外から完了判定できる状態>
- <外から完了判定できる状態>

## 依存関係
- <先に完了または決定しているIssue、仕様、判断>
```

### `child`

```md
## 親Issue
- <親Issue>

## 目的
<このIssueで到達する状態>

## 作業範囲
- <変更対象>
- <変更対象>

## 完了条件
- <外から完了判定できる状態>
- <外から完了判定できる状態>

## 依存関係
- <先に完了または決定しているIssue、仕様、判断>
```

### `parent`

```md
## 目的
<子Issue群で到達する状態>

## 全体の完了条件
- <全体として満たす状態>
- <全体として満たす状態>

## 子Issue
- <子Issue>: <その子Issueが満たす状態>
- <子Issue>: <その子Issueが満たす状態>

## 依存関係
- <先に完了または決定しているIssue、仕様、判断>
```

### 追加項目
検証と参照は、`## 依存関係` の後に置く。

```md
## 検証
- <このIssue固有の確認観点>

## 参照
- <参照元>
```

## 項目の書き方

| 項目 | 書く内容 |
| --- | --- |
| 目的 | そのIssueで到達する状態。背景は目的の理解に必要な範囲に限る。 |
| 作業範囲 | 目的を満たすために変更する対象。実装順ではなく変更単位で整理する。 |
| 完了条件 | 外から完了判定できる条件。手順ではなく成立すべき状態を書く。 |
| 依存関係 | 先に完了または決定している必要があるIssue、仕様、判断。 |
| 検証 | そのIssue固有の確認観点、専用テスト、手動確認。 |
| 参照 | 参照元。 |

作業範囲は、作業手順ではなく変更対象として書く。

## 作成と更新
Issueを作成または更新する前に、タイトル、本文、依存関係をユーザーへ示す。
GitHub操作は `gh` を使う。

Issueを作成または更新した場合は、Issue番号とURLを確認する。

## 報告
本文のみを作成した場合は、タイトル案とIssue本文を返す。
Issueを作成または更新した場合は、Issue番号、タイトル、URLを返す。
入力が不足または矛盾する場合は、不足項目または矛盾箇所を返す。

## Definition of Done
- 本文が、文章前提、Issue種別、本文形式に従っている。
- 目的、作業範囲または全体の完了条件、依存関係が矛盾していない。
- 検証と参照は、そのIssue固有の情報がある場合だけ含まれている。
- 起票または更新した場合は、Issue番号とURLを確認している。
