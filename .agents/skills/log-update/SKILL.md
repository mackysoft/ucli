---

# 目的
- 「いつ」「何を」「なぜ」「どう検証し」「次に何をするか」を短時間で再現できるログを残す。
- 作業の節目での更新を機械化し、ログ未更新の再発を止める。
- spec drift / 分割漏れ / 検証漏れが起きたとき、後から原因追跡できる粒度を確保する。

# 非目的
- 日記や実況（価値の低い逐次記録）は残さない。
- 仕様書（spec）やPR本文の代替をしない（ただしリンク・要点は書く）。
- 機密情報（トークン・鍵・個人情報）を記録しない。

# 使うタイミング
- 作業開始時（作業ブロックの開始）
- 方針・設計判断が確定した
- 主要な変更を入れた
- 検証を走らせた
- 失敗・詰まり・原因調査をした
- コミットを作った
- PRを作る直前（要点・リスク・検証結果のサマリを残す）
- 完了判断をした

# 出力
- `log.md`に、フォーマットに沿った追記（append-only）

# 手順

## 1. 追記フォーマットを選ぶ
- 作業ブロック：`assets/entry_block.md`
- マイクロ更新：`assets/entry_micro.md`

## 2. 追記（append-only）
- 内容を、選んだフォーマットに沿って `log.md` に追記する。

## 記載スタイル
- 事実 → 判断 → 根拠（最小限）の順
- 「何をしたか」より「なぜそれをしたか」「どう検証したか」を優先
- 未確定は未確定として書く（決定事項にしない）

# Definition of Done
- トリガーに対応するエントリが追記されている
- 機密情報が含まれていない
