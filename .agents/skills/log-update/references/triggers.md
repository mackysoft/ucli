# Log update triggers（更新トリガー）

## Block（作業ブロック）で記録すべき
- 方針が決まった（設計判断、方針転換、分割方針など）
- 主要な変更を入れた（責務変更、契約変更、依存方向変更など）
- 失敗・詰まり・原因調査をした（仮説→検証→結論）
- コミットを作った（コミットの目的と範囲）
- PRを作る直前（PRの要点、リスク、検証結果のサマリ）
- 完了判断をした（残タスク/既知リスクが無いか）

## Micro（短い追記）でよい
- format/lint を実行した（PASS/FAIL、変更が出たか）
- tests/coverage を実行した（範囲、結果、失敗分類）
- 小さな再実行（testfilterなどで再走）

## 訂正/追補（Correction/Addendum）
- 直前のログの誤記に気づいた
- 既存エントリに補足が必要になった（後で判明した原因など）
