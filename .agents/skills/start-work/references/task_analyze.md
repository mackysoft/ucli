# Task Analyze

記録時は `Task Size: Small|Medium|Large` を使用する。

## Small
全てを満たす見込み:
- 仕様（受け入れ条件）の追加・変更が不要
- 公開API・データ形式・レイヤー境界に影響しない
- 変更範囲が局所（目安: 1~2ファイル、概ね50行程度）
- 新規テスト追加が不要（既存テストの実行で担保できる）

## Medium
- Small でも Large でもない

## Large
いずれかに該当:
- レイヤーを跨ぐ変更
- 新規概念導入
- 変更が広い（目安: 10ファイル以上、500行以上、複数モジュール）
- 受け入れ条件が多い、段階的に進める必要がある
