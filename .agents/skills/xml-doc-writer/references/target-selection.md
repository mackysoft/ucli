# 対象選定規則

## 優先順位
1. `public` / `protected` の公開API
2. 外部モジュール境界のインターフェース
3. 非公開でも契約の誤用リスクが高いメンバ

## 非公開メンバを対象にする条件
- 境界処理を担当する。
- 永続化、通信、変換処理を担当する。
- 例外契約が利用側の挙動に影響する。
- 非同期完了条件やキャンセル尊重が利用側前提になる。

## 対象外
- 生成コード。
- 命名以上の契約を提供できない単純アクセサ。
- 実装詳細のみで外部契約に影響しない局所処理。

## 生成コードの見分け方
- ファイル先頭やコメントに source generated の記述がある。
- 自動生成ファイルで、再生成により手動編集が失われる。
